•	Prefix: /api/v1.

o	Error: { success: false, code, message, errors? }
•	Query chung: ?page=1&limit=20&sort=updatedAt:desc&search=keyword
•	Validation: Joi/Zod cho body/params/query; dùng validate(schema) middleware.
•	RBAC: Admin-only cho CRUD; gắn authenticate + authorize(['admin']).
•	SQL: dùng mysql2/pool, prepared statements; soft constraints kiểm tra FK khi xoá/cập nhật.

o	GET /buses/:id, POST /buses, PUT /buses/:id, DELETE /buses/:id
o	Validation: seat_count ≥ 8; plate unique; status ∈ {active, inactive, maintenance}
2.	/api/v1/drivers
o	Fields: id, name, phone, license_no, license_expire_at, status
o	Validation: phone định dạng VN; license_expire_at ≥ today

o	Fields: id, name, grade, class_name, guardian_phone, route_id?

2.	/api/v1/routes/:id/stops
o	Fields: id, route_id, name, lat, lng, order_index
o	Endpoints: list/create/update/delete stop; reorder bằng PATCH /routes/:id/stops/reorder (optional)

3) Công việc chi tiết
A. Auth (M0) — REST
•	Endpoint:
o	POST /api/v1/auth/login → sinh accessToken (JWT) + refreshToken.
o	POST /api/v1/auth/refresh → cấp access mới từ refresh hợp lệ.
o	GET /api/v1/auth/profile → trả thông tin user + role.
o	(Optional) POST /api/v1/auth/logout → revoke refresh (nếu lưu DB).
•	JWT:
o	Access: exp ngắn (15–30 phút); payload có sub, role.
o	Refresh: exp dài (7–30 ngày); lưu DB (bảng refresh_tokens) hoặc hash để có thể revoke.
•	Bảo mật:
o	bcrypt hash mật khẩu, lock account, brute-force protection (rate-limit theo IP/user).
o	CORS đúng origin FE; header Authorization: Bearer.
•	Middleware:
o	authenticate: verify JWT, gắn req.user.
o	authorize(roles: Role[]): chặn nếu req.user.role không thuộc roles.
B. WS Handshake guard
•	Helper verifyWsJWT(handshake.auth.token):
o	Xác thực JWT, từ chối nếu thiếu/hết hạn.
o	Gắn socket.data.user = { id, role } cho downstream (Thắng dùng).
•	(Optional) Rate-limit join/emit theo userId.
C. Envelope, lỗi & validation
•	Dùng Global Error Handler chung:
o	OK: { success: true, data, meta? }
o	Error: { success: false, code, message, errors? }
•	Mã lỗi: AUTH_401, AUTH_REFRESH_401, AUTH_FORBIDDEN_403, VALIDATION_422.
•	Validation (Joi/Zod):
o	login: email/phone + password (độ dài tối thiểu);
o	refresh: refreshToken bắt buộc;
o	profile: không cần body.
D. Reporting (M7) — tối thiểu cho dashboard
•	Endpoint:
o	GET /api/v1/trips/stats?from=YYYY-MM-DD&to=YYYY-MM-DD
	Trả: totalTrips, onTimeTrips, delayedTrips, delayAvgMin.
o	GET /api/v1/buses/stats
	Trả: activeBuses, inactiveBuses, (optional) maintenanceCount.
•	Quy ước:
o	Chỉ read-only, RBAC: admin.
o	Truy vấn SQL có index, tránh full scan.
o	meta gồm khoảng thời gian truy vấn.
